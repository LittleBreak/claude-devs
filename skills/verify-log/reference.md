# 问题记录模板与规范

## 章节模板

添加到 task 文档末尾的章节格式：

```markdown
---

## 问题记录

> 本节记录验证阶段发现的问题及其修复过程。

### 问题 1: [简短标题]

- **关联任务**：[对应的任务编号和名称，如 "2.1 创建 vite.config.js"]
- **发现方式**：[自动检测 / 手动发现]
- **问题描述**：[具体现象，包括错误信息、异常行为等]
- **问题原因**：[根本原因分析]
- **解决方案**：[具体的修复措施]
- **改动范围**：[影响的模块/功能]
- **影响评估**：[无影响 / 低 / 中等 / 高] — [简要说明对稳定性或安全性的影响]
```

## 完整示例

```markdown
---

## 问题记录

> 本节记录验证阶段发现的问题及其修复过程。

### 问题 1: Vite 开发服务器启动报错 - 无法解析 alias

- **关联任务**：2.1 创建 vite.config.js
- **发现方式**：自动检测
- **问题描述**：运行 `pnpm dev` 后，Vite 报错 `Failed to resolve import "@/utils/request"`，所有使用 `@` 别名的导入均无法解析。
- **问题原因**：`vite.config.js` 中 `resolve.alias` 配置使用了相对路径 `"src"` 而非绝对路径，Vite 要求 alias 值必须是绝对路径。
- **解决方案**：将 alias 值改为 `path.resolve(__dirname, "src")`，并确认 `path` 模块已正确导入。
- **改动范围**：构建配置模块（vite.config.js）
- **影响评估**：低 — 仅影响开发构建配置，不涉及业务逻辑，无安全性影响。

### 问题 2: Vue 组件热更新失效

- **关联任务**：3.2 配置 HMR
- **发现方式**：手动发现
- **问题描述**：修改 `.vue` 文件后页面未自动刷新，控制台无 HMR 相关日志输出。
- **问题原因**：`@vitejs/plugin-vue2` 未在 `vite.config.js` 的 `plugins` 数组中正确注册，插件函数未被调用（缺少 `()`）。
- **解决方案**：将 `plugins: [vue]` 修改为 `plugins: [vue()]`，确保插件实例正确初始化。
- **改动范围**：构建配置模块（vite.config.js）、开发体验
- **影响评估**：低 — 仅影响开发环境热更新，不影响生产构建和运行时行为。
```

## 影响评估标准

| 等级 | 含义 | 示例 |
|------|------|------|
| 无影响 | 修复不改变任何运行时行为 | 修复注释、文档更新 |
| 低 | 仅影响开发体验或构建配置 | alias 配置修复、HMR 修复 |
| 中等 | 影响部分业务功能或性能 | API 接口调整、路由配置变更 |
| 高 | 影响核心功能、数据安全或系统稳定性 | 认证逻辑修复、数据库迁移修正 |

## 编号规则

- 问题编号在单个 task 文档内递增：问题 1、问题 2、问题 3...
- 如果文档中已有问题记录，新问题编号 = 已有最大编号 + 1
